# X リプライ作成スキル

渡された文章（note記事、Xポスト、ブログ等）を読み、質の高いリプライ案を3つ提案する。

## リプライの設計原則

### アルゴリズム的な価値
- リプライはいいねの**13.5倍**の重み
- 相手が返信すれば**会話成立で75〜150倍**
- つまり「返信が返ってくるリプ」を作ることが最重要

### 良いリプの条件
1. **相手の内容を具体的に参照している**（ちゃんと読んだことが伝わる）
2. **自分の体験や一次情報を添えている**（「この人は実際にやってる人」と認識される）
3. **質問で締めている**（会話が続く確率が上がる）
4. **営業感がゼロ**（ツールや商品のリンクは絶対に入れない）

### 悪いリプの特徴（避けること）
- 「いいですね！」「参考になります！」だけの薄いリプ
- 自分のサービスの宣伝を混ぜる
- 記事を読んでいないのがバレる抽象的な感想
- 「いいね＋リプ」を促すエンゲージメントファーミング

## リプの型（3パターン）

### 型A: 一次情報を添える型（最強）
```
[相手の具体的なポイントに触れる] + [自分の実体験・検証データ] + [質問]
```
- 相手のポストの中で特に刺さったポイントを1つ選ぶ
- そこに対して「自分もこういう経験がある」「実際にやったらこうだった」と一次情報を添える
- 質問で締める

### 型B: 共感+深堀り型
```
[共感] + [なぜ共感したか具体的に] + [さらに聞きたいこと]
```
- 自分が同じ課題を持っていた、同じことで悩んでいたという共感
- 抽象的な「わかります」ではなく、具体的なエピソードを添える
- 「もう少し詳しく聞きたい」という形で質問

### 型C: 別角度からの視点型
```
[相手の主張を認める] + [別の角度からの補足・発見] + [意見を求める]
```
- 相手の内容に反論するのではなく、別の角度から補足する
- 「こういうケースもありますよね」という形
- 「どう思いますか？」で相手の意見を求める

## 出力形式

案ごとに以下を出す:

```
**案A: [型の名前]**
[リプ本文]

**案B: [型の名前]**
[リプ本文]

**案C: [型の名前]**
[リプ本文]
```

最後に「おすすめはX。理由: ...」で推薦する。

## バズ意識（x-buzz-research/analysis.md 準拠）

- リプライはいいねの**13.5倍**の重み。相手が返信すれば**75〜150倍**
- 100-250字 / 3-7行が最もエンゲージメントが高い
- 数字（金額・時間・人数）を入れると保存率UP
- 「知ってた？」型の情報提供や一次情報添えが特に効く
- 質問で締めると会話継続率が上がる → アルゴリズムに乗りやすい

## 注意事項

- リプは**200文字以内**が理想。長すぎると読まれない
- 相手のフォロワー規模に合わせてトーンを調整する（大手にはやや丁寧に、同規模にはフランクに）
- 相手が発信者/インフルエンサーの場合、「教えてください」系の質問が返信を引き出しやすい
- 相手が同規模のセラーの場合、「自分もそうでした」系の共感が関係構築に効く
- **宣伝・営業は一切入れない**。プロフィールを見に来てくれれば十分
