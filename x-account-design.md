# X（旧Twitter）アカウント設計スキル

プロジェクトの目的に合わせたXアカウントの設計と投稿戦略を立案する。以下の知見を適用すること。

---

## 1. アルゴリズムの基本構造（2026年版）

### ランキング要素
- **関連性スコア（40%）**: タイトル・本文のキーワードと検索クエリのマッチ度。Grok AIが文脈・同義語まで理解
- **パフォーマンス指標（30%）**: 販売速度、コンバージョン率、クリック率
- **セラー/アカウント品質（20%）**: 評価、応答速度、違反歴
- **新鮮度（10%）**: 新規投稿への一時的ブースト

### エンゲージメントのスコアリング（いいね=1x基準）
| アクション | 重み |
|-----------|------|
| リプライへの返信（会話成立） | **75〜150x** |
| リツイート | **20x** |
| 引用ポスト | **15x** |
| リプライ | **13.5x** |
| プロフィールクリック | **12x** |
| リンククリック | **11x** |
| ブックマーク | **10x** |
| いいね | **1x** |

**最重要**: 会話成立（リプに返信が付くこと）がいいねの75〜150倍。投稿の最後に質問を入れてリプを誘発し、来たリプには必ず返信する。

### 拡散の流れ
フォロワー/クラスター内の一部に表示 → 初動のエンゲージメント率で判定 → クラスター内拡大 → 隣接クラスターへ

## 2. クラスター概念

### クラスターとは
似た興味関心を持つユーザーの集合体。サービス提供者とサービス利用者は同じクラスター内に混在する。

### なぜ重要か
- 同じポストでも、属するクラスターによってインプレッションが桁違いに変わる
- アルゴリズムは「誰からの反応か」を強く見る。無関係なクラスターからのエンゲージメントはマイナスに働く
- リアル店舗の「立地」と同じ。良い商品でも間違った場所では売れない

### クラスター構築の手順
1. **発信テーマを1つに特化する**（「副業」ではなく「Shopee越境ECの出品自動化」レベルまで絞る）
2. **ターゲットのペルソナを具体化する**（年齢・職業・悩みまで定義）
3. **同属性アカウントをフォローする**（1日10〜20人。同テーマの発信者とそのフォロワー）
4. **同クラスターのポストに反応する**（いいね・リプで「自分はこのクラスターです」とアルゴリズムにシグナルを送る）
5. **無関係なフォローを整理する**（アカウント属性が曖昧になるのを防ぐ）

## 3. 投稿形式の使い分け

### X Articles（記事機能）
- **Premium加入者のみ**作成可能（Basic以上）
- 最大**25,000文字**。見出し・太字・リスト・画像・動画・埋め込み対応
- **投稿後に編集可能**（通常ポストにはない）
- トップパフォーマンス投稿の**45%が記事形式**（2026年初頭データ）
- 通常ポストと違い**2〜3日にわたって波状的に拡散**される
- **ストック型コンテンツ**として蓄積される
- 記事公開後に要約を通常ポストで別途投稿すると認知が広がる
- プロフィールにピン留めしてエバーグリーンコンテンツとして活用

**活用場面**: 詳細ガイド、データ付きレポート、ハウツー、比較分析

### 通常ポスト + セルフリプ
- 本文で興味を引き、セルフリプで**詳細データ・具体例・リンク**を追加
- リンクは**本文に直貼りしない**（最大40%減点）。セルフリプに配置
- セルフリプすると元のポストが**タイムライン上部に再浮上**する
- セルフリプは**2〜3個まで**（多すぎるとスパム感）

**活用場面**: 日常の情報発信、Tips、問題提起、Before/After

### スレッド
- 各ポストが独立したエンゲージメント機会
- 即時のタイムライン発見性ではArticleより優れている

**活用場面**: ストーリーテリング、ライブ実況、ステップバイステップ解説

## 4. 投稿作成のフレームワーク

### 素材と味付け
- **素材**: クラスター内の人たちが興味ある話題か
- **味付け**: 共感・面白さ・有益さなどの切り口
- 有益情報でもクラスターが興味ない素材なら反応は鈍い

### 投稿の型
1. **問題提起→解決策**: 共感を得てからソリューションを提示
2. **数字で驚かせる**: 「公式は5%と言ってますが実際は20%です」
3. **一次情報**: 「実際にやってみた結果」「検証データ」（最もアルゴリズムに評価される）
4. **Before/After**: ビジュアルな比較で変化を見せる
5. **質問形式**: リプを誘発して会話成立を狙う

### 投稿構造
```
本文: 興味を引く導入 → 核心の情報 → 質問で締める
セルフリプ①: 詳細データ・具体例
セルフリプ②: 補足情報 or リンク
```

### 質問の入れ方（会話設計）
- 「○○の経験ある方いますか？」
- 「あなたはどうしてますか？」
- 「一番苦労したポイント、教えてください」
- 「どれが一番参考になりました？」

## 5. 投稿ルール

### 頻度・タイミング
- **1日1〜2ポスト**（6時間以上間隔を空ける）
- 朝（7-8時）or 夜（20-21時）の**固定時間**
- 連続ポストはインプレッション分散を招く（おすすめTLで同一アカウントの重複表示制限）
- X Articles は**週1本**ペース

### 必ず守ること
- **1ポスト1メッセージ**
- **日本語のみ**で書く（日英混在はデブーストされる）
- **ハッシュタグは最大2個**（4つ目から検索非表示）
- **リンクは本文に直貼りしない**（セルフリプに配置）
- **冒頭で興味を引く**（最初の15文字が勝負）
- **リプが来たら必ず返信する**

## 6. やってはいけないこと

### 凍結・シャドウバンリスク
1. **エンゲージメントファーミング**: 「いいねしてね」「RTお願い」
2. **大量フォロー&アンフォロー**: 1日10〜20フォローまで
3. **攻撃的なポストへの関与**: スルー一択
4. **無関係なトレンド便乗**: テーマ外のトレンドに乗らない

### パフォーマンス低下
5. **日英混在ポスト**: 言語不明判定でデブースト
6. **ハッシュタグ3つ以上**: 4つ目から検索非表示
7. **発信テーマと無関係なポスト**: アカウント属性が曖昧になる
8. **URLのみのポスト**: テキスト・画像で補足必須
9. **闇雲な相互交流**: 関連性のないクラスターからのエンゲージメントはマイナス
10. **目的なきフォロワー増加**: フォロワー数よりも質（関連性）

## 7. 成長フェーズ

### Phase 1: 0→100フォロワー（1〜2週間）
- プロフィール整備（テーマキーワードを含める）
- 同クラスターアカウントを1日10〜15フォロー
- 毎日1〜2ポスト + クラスター内リプ3〜5件
- スペース参加（声を上げる場が少ない段階で目立てる）

### Phase 2: 100→500フォロワー（1〜2ヶ月）
- 一次情報の継続投稿
- X Articlesの活用開始
- 伸びたポストの型を分析→横展開
- 同クラスター内での認知を積み上げる

### Phase 3: 500→1000フォロワー（2〜3ヶ月）
- 「この人といえば○○」のポジション確立
- ツール/サービスの直接訴求を増やしてOK（20%程度まで）
- note等のストック型コンテンツとの相互導線が回り始める

### 最初の1000人の重要性
フォロワーの属性がアカウント全体の属性をアルゴリズムに決定づける。関連性の低いフォロワーで固めると将来の伸びに悪影響。

## 8. X Premium

### 加入推奨
- Premium加入者は**2〜4倍のリーチブースト**
- **記事機能**が使える（最大25,000文字、書式設定、画像埋め込み）
- **アナリティクス**で投稿のパフォーマンスを詳細に分析可能
- 月額368円（Basic）からなのでROIは非常に高い

### プラン選択の目安
- **Basic（368円）**: 記事機能を使いたいだけならこれで十分
- **Premium（980円）**: アナリティクス + 認証バッジ + 収益化
- **Premium+（1,960円）**: 広告非表示が必要な場合

## 9. 他プラットフォームとの連携

### noteとのフライホイール
```
noteで長文記事を公開
  ↓
Xで要約を通常ポスト（リンクはセルフリプに）
  ↓
X経由でnote閲覧数増加
  ↓
note内のリンクからサービス認知
  ↓
利用者がXでシェア → 新規フォロワー獲得 → ループ
```

### X Articles + noteの使い分け
- 同じ内容をX ArticlesとnoteのDouble Postで最大リーチ
- X Articlesはアルゴリズム優遇で即時リーチ
- noteはSEOで長期的な検索流入

## 10. KPI

### 週次
- インプレッション数（週合計）
- プロフィール訪問数
- フォロワー増加数
- サイトへのクリック数（UTMパラメータで計測）

### 月次
- フォロワー数の推移
- 投稿あたりの平均エンゲージメント率
- サービスへの流入数・成約数

### 判断基準
- **インプレッション伸びない**: 素材がクラスターに刺さっていない → 伸びているポストを分析
- **インプレッションあるがフォローされない**: プロフィールが弱い → 「何が得られるか」を明確に
- **フォロワー増えるがサービスに来ない**: 訴求頻度が少ない or CTAが弱い

## このスキルの使い方

ユーザーがXアカウントの設計を依頼したら:

1. **ヒアリング**: プロジェクトの目的、ターゲット、提供価値を確認
2. **クラスター設計**: どのクラスターに属するか、フォローすべきアカウントの方針
3. **プロフィール設計**: アカウント名、プロフィール文、キーワード
4. **投稿カテゴリと比率**: Tips / 開発裏話 / トレンド / 直接訴求 の配分
5. **コンテンツカレンダー**: 最初の2週間分の具体的な投稿案（本文+セルフリプ+会話設計）
6. **やってはいけないことリスト**: プロジェクト固有のリスクを追加
7. **成長ロードマップ**: フェーズ別の目標とアクション
