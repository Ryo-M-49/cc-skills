# x-research

X(Twitter)のバズ記事を自動収集し、ビジネス・AI系に絞ったレポートを生成するスキル。

SocialData APIを使って検索し、記事タイトル・本文冒頭・エンゲージメント数を取得する。日本語フィルタとトピックフィルタで関係のない記事を除外する。

## 必要なもの

- Node.js 20+
- SocialData APIキー（https://socialdata.tools/signup で取得）

## セットアップ

ルートの `.env` に APIキーを追加する。

```
SOCIALDATA_API_KEY=your_api_key_here
```

## Claude Codeでの使い方

自然言語で指示するだけで動く。

```
Xのバズ記事をリサーチして
```

条件を指定する場合:

```
Xのバズ記事、いいね500以上で直近1週間
```

```
Xの記事リサーチ、いいね2000以上、直近3日
```

Claude Codeが以下の流れで処理する:

1. `--dry-run` でコスト見積もりを表示
2. ユーザーに確認を求める（APIコストが発生するため）
3. OKが出たら実行
4. レポートをファイル保存

デフォルト設定: いいね1000以上、直近30日、日本語、ビジネス・AI系に自動フィルタ。

## 直接実行する場合

```bash
# 見積もりだけ確認
npx tsx x-research/fetch-x-articles.ts --min-faves 1000 --days 30 --dry-run

# 実行してレポート出力
npx tsx x-research/fetch-x-articles.ts --min-faves 1000 --days 30 --output report.md
```

## オプション

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `--min-faves` | 1000 | 最低いいね数 |
| `--days` | 30 | 直近何日分を検索するか |
| `--lang` | ja | `ja` で日本語のみ、`all` で全言語 |
| `--topic` | (ビジネス・AI系) | カンマ区切りのキーワード。省略時はデフォルトキーワードで自動フィルタ |
| `--max-pages` | 10 | 検索ページ数（1ページ≈20件） |
| `--top-detail` | 30 | 記事タイトル・本文を取得する上位件数 |
| `--output` | (stdout) | 出力ファイルパス |
| `--dry-run` | - | コスト見積もりのみ表示して終了 |

## 仕組み

1. SocialData APIで `url:x.com/i/article min_faves:N` を検索（記事投稿のみ抽出）
2. 日本語フィルタ（ひらがな・カタカナ必須、中国語のみのアカウントを除外）
3. トピックフィルタ（プロフィールのキーワードマッチ）
4. `/twitter/article/{tweetId}` で記事タイトル・本文を取得
5. 記事内容でも再度トピックフィルタ
6. エンゲージメント順にソートしてレポート生成

## レポートに含まれるデータ

各記事ごとに以下が記載される:

- 記事タイトル
- 著者名、ユーザー名、フォロワー数
- いいね数、RT数、閲覧数、ブックマーク数
- 記事URL
- 投稿日
- 本文冒頭200文字
